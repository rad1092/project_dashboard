# 데이터 소스 확장 가이드

## 현재 상태

- 전처리 완료 CSV 사용
- 외부 폴더 읽기 전용
- 실시간 공공 API 미연결

즉, 지금은 "정적 데이터로 설명 가능한 추천 흐름을 유지하는 단계"라고 보면 된다.

## 실시간 특보로 확장할 때

1. `dashboard/services/disaster_data.py` 에 실시간 로딩 함수 추가
2. 기존 `build_alert_summary()` 와 같은 반환 형식을 유지
3. `pages/2_대피소_추천.py` 는 같은 인터페이스로 결과만 받아 쓰게 유지
4. `pages/4_실시간_준비.py` 의 스텁과 docs 를 실제 구현 기준으로 갱신

여기서 중요한 것은 페이지를 먼저 바꾸는 것이 아니라,
기존 서비스 함수와 최대한 비슷한 계약을 새 데이터 소스도 유지하게 만드는 것이다.

## 자동 위치로 확장할 때

1. 페이지 입력 구조는 유지
2. 수동 좌표 입력 대신 브라우저 권한 결과를 세션 상태에 채움
3. 좌표가 들어간 뒤 추천 함수는 그대로 재사용

초보자 기준으로 보면 "좌표를 얻는 방식"만 바뀌고,
그 뒤 감지 지역과 활성 지역 흐름은 최대한 그대로 유지하는 것이 목표다.

## 경로 API 로 확장할 때

1. 현재 직선 거리 계산과 별도 함수로 분리
2. 추천 정렬 기준이 바뀌는지 먼저 결정
3. 유료 API 사용 여부와 비용 정책을 문서화

현재 저장소는 `직선 거리` 를 공통 기준으로 쓰고 있으므로,
실제 경로 API 를 붙이면 아래도 같이 점검해야 한다.

- 카드 문구
- 표의 거리 열
- 지도 설명 문구
- `3 작동 설명` 페이지의 현재/미래 설명
