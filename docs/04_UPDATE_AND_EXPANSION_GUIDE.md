# 업데이트와 확장 가이드

## 텍스트를 바꿀 때

1. `dashboard/content.py` 수정
2. 관련 페이지 문구 수정
3. `README.md` 와 관련 docs 수정
4. 화면에서 흐름 확인

## 추천 규칙을 바꿀 때

1. `dashboard/services/shelter_recommendation.py` 수정
2. `pages/2_대피소_추천.py` 와 `pages/3_작동_설명.py` 설명 확인
3. 테스트 갱신

추천 규칙만이 아니라 추천 페이지 입력 방식이 바뀌면,
현재는 `좌표 -> 자동 감지 지역 -> 활성 지역` 흐름이 유지되는지도 함께 확인해야 한다.

## 데이터 경로를 바꿀 때

1. `PREPROCESSING_DATA_DIR` 또는 secrets 설정 확인
2. `dashboard/services/disaster_data.py` 동작 확인
3. 홈과 추천 페이지에서 로딩되는지 점검

특히 추천 페이지는 통합 대피소 데이터를 이용해 지역 중심 좌표를 계산하므로,
경로 변경 뒤에는 `infer_region_from_coordinates()` 가 정상 동작하는지 같이 확인해야 한다.

## 실시간 기능을 붙일 때

1. `pages/4_실시간_준비.py` 와 docs/15, docs/16 먼저 확인
2. 자동 위치, 실시간 특보, 경로 API 중 무엇을 먼저 붙일지 결정
3. 서비스 계층에 실제 함수 추가
4. 현재 추천 페이지가 그대로 재사용되는지 확인

현재 단계에서는 수동 좌표 입력과 로컬 지역 자동 감지가 함께 동작한다.
실시간 위치 API 를 붙일 때도 추천 페이지 자체를 갈아엎기보다,
좌표를 채우는 방식과 지역 감지 방식을 교체하는 쪽으로 확장하는 것이 안전하다.
