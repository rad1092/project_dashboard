# 업데이트와 확장 가이드

## 이 문서를 읽으면 좋은 사람

- 문구만 살짝 고치려는 사람
- 추천 규칙을 바꾸려는 사람
- 데이터 폴더 위치를 바꾸려는 사람
- 실시간 기능을 어디에 붙여야 할지 찾는 사람

핵심은 "무엇을 바꾸느냐에 따라 같이 확인해야 할 파일이 다르다"는 점이다.

## 텍스트를 바꿀 때

1. `dashboard/content.py` 수정
2. 관련 페이지 문구 수정
3. `README.md` 와 관련 docs 수정
4. 화면에서 흐름 확인

문구 수정이라고 해도 아래는 같이 확인하는 편이 좋다.

- 홈과 About 에서 같은 개념을 다른 말로 쓰지 않는가
- `감지 지역`, `활성 지역`, `직선 거리` 같은 핵심 용어가 문서와 코드에서 같은 뜻으로 쓰이는가
- 현재 구현과 미래 확장을 섞어 쓰지 않았는가

## 추천 규칙을 바꿀 때

1. `dashboard/services/shelter_recommendation.py` 수정
2. `pages/2_대피소_추천.py` 와 `pages/3_작동_설명.py` 설명 확인
3. 테스트 갱신

추천 규칙만이 아니라 추천 페이지 입력 방식이 바뀌면,
현재는 `좌표 -> 자동 감지 지역 -> 활성 지역` 흐름이 유지되는지도 함께 확인해야 한다.

초보자 기준으로는 아래 질문을 같이 점검하면 실수가 줄어든다.

- 전용 대피소와 대체 대피소의 기준이 바뀌었는가
- 추천 정렬 기준이 거리 우선인지 수용인원 우선인지 달라졌는가
- 추천 결과 표와 카드의 설명 문구도 같은 기준을 말하고 있는가

## 데이터 경로를 바꿀 때

1. 저장소 내부 `preprocessing_data` 기본 경로를 먼저 확인
2. 외부 데이터를 쓸 때만 `PREPROCESSING_DATA_DIR` 또는 secrets 설정 확인
3. `dashboard/services/disaster_data.py` 동작 확인
4. 홈과 추천 페이지에서 로딩되는지 점검

특히 추천 페이지는 통합 대피소 데이터를 이용해 지역 중심 좌표를 계산하므로,
경로 변경 뒤에는 `infer_region_from_coordinates()` 가 정상 동작하는지 같이 확인해야 한다.

즉, 경로 변경은 단순히 파일을 읽는지 여부만이 아니라
추천 흐름의 첫 단계인 감지 지역 계산까지 함께 확인해야 하는 작업이다.

## 실시간 기능을 붙일 때

1. `pages/4_실시간_준비.py` 와 docs/15, docs/16 먼저 확인
2. 자동 위치, 실시간 특보, 경로 API 중 무엇을 먼저 붙일지 결정
3. 서비스 계층에 실제 함수 추가
4. 현재 추천 페이지가 그대로 재사용되는지 확인

현재 단계에서는 수동 좌표 입력과 로컬 지역 자동 감지가 함께 동작한다.
실시간 위치 API 를 붙일 때도 추천 페이지 자체를 갈아엎기보다,
좌표를 채우는 방식과 지역 감지 방식을 교체하는 쪽으로 확장하는 것이 안전하다.

가장 안전한 확장 순서는 보통 아래와 같다.

1. 서비스 계층 인터페이스부터 정리
2. 준비 페이지와 문서 설명 갱신
3. 실제 API 연동 추가
4. 추천 페이지와 테스트를 같은 기준으로 맞춤
