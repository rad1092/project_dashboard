# 진입점과 설정 가이드

## 이 문서의 목적

앱이 어디서 시작되고, 공통 설정이 어디에 모여 있는지 빠르게 파악하기 위한 문서다.
처음 보는 사람은 `app.py` 와 `dashboard/config.py` 를 함께 읽으면 전체 구조가 빠르게 잡힌다.

## `app.py`

- 홈 화면 진입점
- 현재 데이터 범위와 페이지 순서를 안내
- 저장소 내부 전처리 데이터 폴더가 실제로 읽히는지 가장 먼저 확인하는 화면

`app.py` 는 실제 추천 계산을 하지 않는다.
대신 홈 화면에서 프로젝트 개요, 연결된 데이터셋, 현재 한계를 먼저 보여 주고,
세부 계산은 서비스 계층에 맡긴다.

## `dashboard/config.py`

- 앱 제목, 아이콘, 페이지 메타데이터 관리
- 페이지별 라벨과 요약을 한곳에서 유지
- 새 페이지를 추가하면 가장 먼저 이 파일을 확인

특히 `PAGE_META` 는 아래를 한 번에 맞추는 기준표다.

- 브라우저 탭 제목
- 홈 화면의 페이지 소개 문구
- 각 페이지 라벨 체계

즉, 페이지 이름이나 순서를 바꿀 때 문자열을 여러 파일에서 따로 고치지 않게 해 준다.

## `.streamlit/config.toml`

- 앱 테마와 브라우저 설정 담당
- 현재는 밝은 테마와 기본 색상만 관리

이 파일은 화면의 색과 기본 동작을 바꾸는 곳이지,
추천 규칙이나 데이터 경로를 바꾸는 곳은 아니다.

## `.streamlit/secrets.toml.example`

- `preprocessing_data_dir` 선택적 override 예시 제공
- 미래 API 키 자리표시자만 포함
- 실제 키는 Git 에 올리지 않음

초보자 기준으로는 아래처럼 이해하면 충분하다.

- 기본 실행: 저장소 내부 `preprocessing_data`
- 다른 데이터 위치 사용: `PREPROCESSING_DATA_DIR` 또는 secrets 로 override
- 실제 비밀값: 예시 파일이 아니라 로컬 전용 secrets 파일에 따로 둠
