# 진입점과 설정 가이드

## 설명 대상 파일

- `README.md`
- `pyproject.toml`
- `.gitignore`
- `.streamlit/config.toml`
- `.streamlit/secrets.toml.example`
- `app.py`
- `dashboard/config.py`

## `README.md`

### 왜 있는가

저장소를 처음 보는 사람이 목적과 문서 지도를 가장 먼저 이해하게 한다.

### 언제 쓰는가

- 저장소 소개를 바꿀 때
- 문서 목록이 늘거나 줄 때
- 실행 방법이 바뀔 때

### 현재 코드 예시

```md
Streamlit을 이용해 프로젝트 결과, 데이터 분석 과정, 학습 내용을 정리하고 설명하는 저장소입니다.
```

### 수정 예시

- 문구가 너무 기능 중심이면 저장소 목적 중심으로 다시 쓴다.
- 새 문서를 추가했다면 문서 목록에 역할까지 같이 적는다.

### 주의점

- README가 앱 화면 설명만 하고 저장소 목적을 설명하지 않으면 방향이 흐려진다.

## `pyproject.toml`

### 왜 있는가

실행 환경과 의존성 기준을 관리한다.

### 언제 쓰는가

- 새 패키지를 추가할 때
- 버전 충돌을 해결할 때
- 프로젝트 설명 메타데이터를 바꿀 때

### 현재 코드 예시

```toml
[dependency-groups]
dev = [
    "pytest>=8.4.2",
]
```

### 수정 예시

- API 클라이언트용 패키지를 추가해야 하면 여기 먼저 넣고 lock을 맞춘다.

### 주의점

- Streamlit, pandas 같은 핵심 패키지는 호환 버전을 먼저 확인한다.
- 설명만 바꾸고 lock을 갱신하지 않으면 환경 재현이 어긋날 수 있다.

## `.gitignore`

### 왜 있는가

로컬 캐시, secrets, 빌드 산출물을 Git 추적에서 제외한다.

### 언제 쓰는가

- 새 캐시 디렉터리나 생성 산출물이 생길 때

### 주의점

- `.streamlit/secrets.toml`이 빠지면 비밀정보 노출 위험이 생긴다.

## `.streamlit/config.toml`

### 왜 있는가

앱 테마와 Streamlit 기본 옵션을 맞춘다.

### 언제 쓰는가

- 색상, 폰트, 통계 옵션을 바꿀 때

### 주의점

- TOML 문법과 인코딩이 틀리면 앱 시작 시 경고가 발생한다.

## `.streamlit/secrets.toml.example`

### 왜 있는가

실제 secrets 파일 구조를 템플릿으로 설명한다.

### 언제 쓰는가

- API 키나 DB URL 구조가 바뀔 때

### 주의점

- 예시 파일만 버전관리하고 실제 secrets 파일은 올리지 않는다.

## `dashboard/config.py`

### 왜 있는가

앱 제목, 아이콘, 각 페이지의 공통 메타를 한 곳에서 관리한다.

### 현재 코드 예시

```python
APP_TITLE = "Streamlit Project Workspace"
```

### 수정 예시

- 홈 화면 이름을 바꾸고 싶으면 먼저 `PAGE_META["home"]`를 바꾼다.

### 주의점

- 페이지 요약과 실제 페이지 내용이 다르면 홈 화면 안내가 틀어진다.

## `app.py`

### 왜 있는가

저장소 홈 화면이며, 앱의 첫 인상과 문서 진입점을 제공한다.

### 현재 코드 예시

```python
dataframe = load_demo_dataset()
kpis = build_kpis(dataframe)
```

### 수정 예시

- 홈 화면에서 새 요약 카드를 보여주고 싶으면 서비스 함수 반환값을 먼저 확인한다.
- 새 문서를 추가했다면 오른쪽 안내 박스 문구도 함께 갱신한다.

### 주의점

- 홈 화면 안에서 직접 API 호출이나 무거운 데이터 준비를 하지 않는다.
- 홈 문구가 README와 다른 이야기를 하지 않게 맞춘다.

## 잘못 수정하면 생길 문제

- 설정 파일 인코딩이 깨지면 앱 부팅 경고가 난다.
- `pyproject.toml` 버전 제약이 잘못되면 lock과 실행이 깨진다.
- README와 홈 화면 문구가 다르면 저장소 정체성이 흔들린다.
