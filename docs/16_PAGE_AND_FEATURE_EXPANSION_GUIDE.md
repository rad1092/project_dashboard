# 페이지와 기능 확장 가이드

## 이 문서의 목적

새 페이지, 새 탭, 새 카드, 새 필터, 새 차트 같은 기능을 추가할 때 현재 구조를 어떻게 유지하면서 확장할지 설명하는 문서다.

## 먼저 판단할 것

### 새 페이지가 필요한 경우

- 목적이 기존 페이지와 분명히 다를 때
- 화면 흐름이 너무 길어져 한 페이지에서 읽기 어려울 때
- 별도 문서와 설명이 필요할 정도로 독립적인 기능일 때

### 기존 페이지 확장으로 충분한 경우

- 같은 데이터와 같은 맥락 안에서 필터나 차트만 조금 늘어나는 경우
- 기존 페이지의 하위 탭으로 자연스럽게 들어갈 수 있는 경우

## 새 페이지를 추가할 때

### 순서

1. `pages/` 아래에 번호가 붙은 새 파일 생성
2. `apply_page_config()` 호출
3. 필요한 콘텐츠를 `content.py`에 추가
4. 반복 UI가 있으면 `components/`로 분리
5. README와 docs 갱신

### 주의점

- 새 페이지가 생기면 구조 설명 문서도 같이 바꿔야 한다.
- 페이지 파일이 화면과 데이터 로직을 동시에 과하게 떠안지 않게 한다.

## 새 카드나 새 섹션을 추가할 때

### 순서

1. 공통 데이터인지 확인
2. 반복될 표현인지 확인
3. 반복되면 컴포넌트 함수로 분리
4. 한 페이지 전용이면 페이지 안에 유지

### 주의점

- 같은 카드 패턴을 여러 페이지에 복붙하지 않는다.
- 표시만 바꾸는 작업인지, 데이터 구조 자체가 필요한 작업인지 먼저 구분한다.

## 새 필터를 추가할 때

### 순서

1. 해당 필터용 컬럼이 있는지 확인
2. 서비스 계층 반환 구조 점검
3. 페이지 사이드바에 필터 추가
4. KPI, 차트, 표가 같은 필터를 공유하게 연결

### 주의점

- 필터가 많아질수록 rerun 비용과 코드 복잡도가 올라간다.
- 일부 차트만 다른 필터를 쓰게 만들면 해석이 어려워진다.

## 새 차트를 추가할 때

### 순서

1. 차트가 답할 질문 정의
2. 필요한 집계 로직 정의
3. `charts.py`에 함수 추가
4. 페이지에서 호출

### 주의점

- 차트 함수 안에 외부 요청을 넣지 않는다.
- 페이지마다 같은 집계를 반복하지 않는다.

## 상태관리나 상호작용 기능을 추가할 때

### 주의점

- Streamlit의 rerun 특성을 먼저 이해한다.
- `session_state`를 남용하면 흐름이 복잡해진다.
- 상태가 정말 필요한 경우에만 최소 범위로 쓴다.
