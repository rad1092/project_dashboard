# 프로젝트 설명 가이드

## 언제 쓰는 문서인가

- README 소개 문장을 정리할 때
- 발표 자료나 문서 첫 장 문구를 쓸 때
- 프로젝트를 다른 사람에게 짧게 소개해야 할 때

## 짧게 설명할 때

- 전처리된 재난 특보와 대피소 데이터를 이용해 대피소 추천과 분석 구조를 함께 보여주는 Streamlit 프로젝트입니다.
- 현재는 과거 데이터 기반으로 동작하고, 실시간 확장 위치를 코드와 문서에 먼저 정리해 두었습니다.

## 조금 더 길게 설명할 때

이 프로젝트는 단순히 추천 결과만 보여주는 것이 아니라, 어떤 데이터와 규칙으로 결과가 나왔는지 같이 설명하는 구조를 목표로 한다. 그래서 추천 페이지, 작동 설명 페이지, 실시간 준비 페이지, 분석 페이지를 분리해 현재 구현과 미래 확장을 한 저장소 안에서 함께 보여준다.

조금 더 풀어 말하면 다음을 강조할 수 있다.

- 추천 결과와 그 근거를 같은 저장소 안에서 함께 관리한다.
- 감지 지역과 활성 지역 개념을 분리해 현재 위치 기반 추천 흐름을 설명 가능하게 만든다.
- 실시간 기능은 아직 실행하지 않지만, 어디에 붙일지는 코드와 문서에 먼저 남겨 둔다.

## 강조하면 좋은 포인트

- 유료 API 없이도 추천 흐름을 설계했다.
- 외부 전처리 CSV 를 읽기 전용으로 두고 앱 구조를 분리했다.
- 추천 결과와 설명 문서를 같이 유지한다.
- 실시간 확장 위치를 별도 페이지와 주석으로 남겨 두었다.

## 피하면 좋은 표현

- "실시간 대피 안내 앱"처럼 현재보다 과장된 표현
- "정확한 위치 판별"처럼 행정경계 기반 역지오코딩을 하는 것처럼 들리는 표현
- "최단 경로 안내"처럼 실제 경로 API 가 붙어 있는 것처럼 보이는 표현

현재 단계의 정확한 설명은 "과거 전처리 데이터와 직선 거리 기준 추천"이다.
