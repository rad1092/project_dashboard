# 평면 구조 개요

현재 저장소는 `app.py + pages/*.py` 중심 구조다.

## 핵심 원칙

- 홈은 `app.py`가 단독으로 책임진다.
- 각 페이지는 자기 화면에서 쓰는 상수, 데이터 로더, 계산 로직, UI helper 를 파일 내부에 직접 둔다.
- 페이지끼리 import 하지 않는다.
- 공용 계층을 감추는 대신, 중복을 허용하고 파일 단위 가독성을 우선한다.

## 읽는 순서

1. `app.py`
2. `pages/2_대피소_추천.py`
3. `pages/6_Data_Analysis.py`
4. 나머지 정적 설명 페이지

## 왜 이렇게 바꿨는가

이전 구조는 화면, 서비스, 콘텐츠, 컴포넌트가 여러 단계로 나뉘어 있어
초보자가 실제 흐름을 따라가기 어려웠다.

지금은 페이지 파일 하나를 열면 아래를 한 번에 볼 수 있다.

- 어떤 문구를 쓰는가
- 어떤 CSV 를 읽는가
- 어떤 계산을 하는가
- 어떤 UI 를 그리는가
