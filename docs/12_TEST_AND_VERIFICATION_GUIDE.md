# 테스트와 검증 가이드

## 이 문서의 목적

자동 테스트와 수동 검증이 각각 무엇을 확인하는지 정리한 문서다.
초보자라면 "테스트는 숫자만 맞는지 보는 것이 아니라, 저장소 안의 약속을 보호한다"는 점을 기억하면 좋다.

## 자동 테스트

```powershell
.venv\Scripts\python.exe -m pytest
```

## 현재 테스트 파일

- `tests/test_analysis_data.py`: 분석용 특보 DataFrame 과 KPI 검증
- `tests/test_content.py`: 콘텐츠 구조와 페이지 문법 검증
- `tests/test_disaster_data.py`: 경로 탐색, CSV 로딩, 최근 특보 요약 검증
- `tests/test_shelter_recommendation.py`: 재난 정규화, 거리 계산, 추천 규칙 검증

각 테스트 파일은 아래 층을 보호한다고 보면 이해가 쉽다.

- `conftest.py`: 임시 전처리 데이터 준비
- `test_disaster_data.py`: 데이터 서비스 계약
- `test_shelter_recommendation.py`: 추천 규칙 계약
- `test_analysis_data.py`: 분석용 요약 계약
- `test_content.py`: 설명 데이터와 페이지 문법 계약

## 수동 검증

1. `Home` 에서 데이터 로딩과 페이지 안내가 보이는지 확인
2. `2 대피소 추천` 에서 좌표 입력, 추천 카드, 지도, 표가 함께 보이는지 확인
3. `3 작동 설명` 이 이미지 없이도 플로우를 설명하는지 확인
4. `4 실시간 준비` 가 실제 호출 없이 준비 페이지로 읽히는지 확인
5. `6 Data Analysis` 차트와 표가 필터에 맞게 바뀌는지 확인

수동 검증에서 특히 중요한 질문은 아래와 같다.

- 좌표를 바꾸면 감지 지역과 활성 지역 설명이 자연스럽게 따라오는가
- 수동 지역 보정을 켜면 추천 결과와 특보 요약이 같은 지역을 바라보는가
- 카드, 표, 지도에서 추천 순서가 서로 다르게 보이지 않는가

## 특히 확인할 것

- 외부 CSV 를 수정하지 않았는가
- 유료 API 호출 코드가 들어가지 않았는가
- 새 Python 파일과 함수에 설명 주석이 충분한가
- 현재 구현과 미래 확장을 섞는 문구가 새로 생기지 않았는가
