# 데이터 흐름

## 데이터 폴더를 찾는 순서

`dashboard_data.py`의 `resolve_data_dir()`는 아래 순서대로 전처리 데이터 폴더를 찾는다.

1. 함수 인자로 직접 넘긴 경로
2. 환경변수 `PREPROCESSING_DATA_DIR`
3. `.streamlit/secrets.toml`의 `preprocessing_data_dir`
4. 저장소 내부 `preprocessing_data/`
5. 예전 호환용 `~/Desktop/preprocessing_data`

이 순서를 둔 이유는, 테스트나 배포 환경에서는 강제로 경로를 바꾸기 쉽고, 일반 실행에서는 저장소 안 기본 데이터를 바로 읽게 하려는 것이다.

## CSV가 공통 스키마로 바뀌는 과정

### 특보 데이터

- 원본 파일: `preprocessing/danger_clean.csv`
- 핵심 컬럼: `발표시간`, `지역`, `시군구`, `재난종류`, `특보등급`
- 추가 정리:
  - `발표시간`을 날짜형으로 변환
  - `시군구정규화`를 만들어 `"포항시"`와 `"포항"`처럼 표기 차이를 비교하기 쉽게 맞춤

### 통합 대피소 데이터

- 원본 파일: `preprocessing/final_shelter_dataset.csv`
- 핵심 컬럼: 이름, 주소, 유형, 위도, 경도, 시도, 시군구, 수용인원
- 추가 정리:
  - 위도/경도/수용인원을 숫자로 변환
  - `시군구정규화`, `수용인원_정렬값`을 추가

### 전용 대피소 데이터

- 원본 파일:
  - `preprocessing/earthquake_shelter_clean_2.csv`
  - `preprocessing/tsunami_shelter_clean_2.csv`
- 목적:
  - 추천 화면에서 지진, 해일/쓰나미처럼 전용 대피소가 필요한 재난을 우선 처리하기 위해 따로 읽는다.
- 추가 정리:
  - 일반 대피소와 비슷한 컬럼 구조로 맞춰서 추천 함수가 같은 방식으로 계산할 수 있게 한다.

## 화면별 사용 방식

### 홈 `app.py`

- 공용 데이터 로더를 호출해 현재 데이터 건수와 최근 특보 시각을 보여준다.
- 여기서는 계산 로직을 만들지 않고 요약 값만 보여준다.

### 추천 `pages/1_대피소_추천.py`

- 현재 좌표로 가장 가까운 지역 중심을 추정한다.
- 최근 특보 이력과 기본 재난 목록을 합쳐 사용자가 고를 재난 옵션을 만든다.
- 재난 유형에 따라 전용 대피소를 먼저 찾고, 부족하면 통합 대피소를 fallback으로 합친다.

### 실시간 `pages/2_실시간_테스트.py`

- 추천 페이지 helper를 재사용해서 "지역 추정 -> 추천 계산"은 같은 규칙으로 유지한다.
- 추가로 브라우저 위치와 OSRM 도보 경로를 붙인다.
- OSRM이 실패하면 직선 fallback 경로를 만들어 화면이 비지 않게 한다.

### 분석 `pages/3_Data_Analysis.py`

- 특보 데이터를 다시 분석용 DataFrame으로 만들고 `재난그룹` 컬럼을 추가한다.
- 추천 화면처럼 한 지역의 결과를 뽑는 대신, 전체 분포와 추이를 읽는 데 집중한다.
