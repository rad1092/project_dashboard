# 데이터 흐름

## 데이터 소스

- 정적 CSV
  - `preprocessing_data/preprocessing/danger_clean.csv`
  - `preprocessing_data/preprocessing/final_shelter_dataset.csv`
  - `preprocessing_data/preprocessing/earthquake_shelter_clean_2.csv`
  - `preprocessing_data/preprocessing/tsunami_shelter_clean_2.csv`
- 실시간 재난문자 CSV
  - `preprocessing_code/data/disaster_message_realtime.csv`
  - 필요 시 `preprocessing_code/mock_disaster_message.py`로 모의 재난문자를 생성해 같은 경로를 덮어쓸 수 있다.

## 화면별 사용 방식

### `app.py`

- HOME 랜딩에서 전처리 CSV를 읽어 KPI와 최신 특보 시각을 요약한다.
- `pages/3_데이터_분석.py`가 재사용하는 공용 CSV 로더를 제공한다.

### `pages/1_대피_안내_시뮬레이션.py`

- 정적 특보 CSV와 대피소 CSV를 기반으로 현재 위치와 재난 유형에 맞는 대피 경로를 계산한다.

### `pages/2_실시간_대피_안내.py`

- 실시간 재난문자 CSV를 읽고 감지 지역 기준 최신 문자 1건을 선택해 대피소 추천과 경로 안내를 만든다.

### `pages/3_데이터_분석.py`

- `danger_clean.csv`와 `final_shelter_dataset.csv`를 다시 집계해 재난 분포, 시계열 추이, 지역 비교, 대피소 연계를 시각화한다.
- 하단 `원본 데이터` expander에서는 필터가 적용된 특보 원본 테이블을 그대로 보여준다.

### `pages/4_권역_대피소_지도.py`

- 생성된 HTML 지도를 그대로 불러와 전체 대피소 위치를 보여준다.
