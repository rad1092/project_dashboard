# 데이터 흐름

## 데이터 폴더를 찾는 순서

`dashboard_data.py`의 `resolve_data_dir()`는 아래 순서대로 전처리 데이터 폴더를 찾는다.

1. 함수 인자로 직접 넘긴 경로
2. 환경변수 `PREPROCESSING_DATA_DIR`
3. `.streamlit/secrets.toml`의 `preprocessing_data_dir`
4. 저장소 내부 `preprocessing_data/`
5. 예전 호환용 `~/Desktop/preprocessing_data`

이 순서를 둔 이유는 테스트나 배포 환경에서는 경로를 강제로 바꾸기 쉽고, 일반 실행에서는 저장소 기본 데이터를 바로 읽게 하려는 것이다.

## CSV가 공통 스키마로 바뀌는 과정

### 기존 특보 데이터

- 원본 파일: `preprocessing/danger_clean.csv`
- 담당 모듈: `dashboard_data.py`
- 공통 컬럼: `발표시각`, `지역`, `시군구`, `재난종류`, `특보등급`, `해당지역`
- 추가 정리:
  - `발표시각`을 datetime으로 변환
  - `시군구정규화`를 만들어 `포항시`와 `포항` 같은 표기 차이를 비교하기 쉽게 맞춤

### 통합/전용 대피소 데이터

- 원본 파일:
  - `preprocessing/final_shelter_dataset.csv`
  - `preprocessing/earthquake_shelter_clean_2.csv`
  - `preprocessing/tsunami_shelter_clean_2.csv`
- 담당 모듈: `dashboard_data.py`
- 목적:
  - 추천 페이지와 실시간 안내 페이지가 같은 추천 스키마를 쓰도록 맞춘다.

### 크롤링 재난문자 데이터

- 원본 파일: `preprocessing_code/data/disaster_message_realtime.csv`
- 담당 모듈: `crawler_alerts_data.py`
- 원본 컬럼: `발표시각`, `지역`, `시군구`, `재난종류`, `특보등급`, `내용`, `발송기관`, `번호`
- 추가 정리:
  - `발표시각`을 datetime으로 변환
  - `시군구정규화` 추가
  - `재난종류`를 `재난그룹`으로 매핑
  - `alert_key`를 만들어 최근 재난문자 선택 UI에서 안정적으로 사용

## 화면별 사용 방식

### 홈 `app.py`

- 공용 데이터 로더를 호출해 현재 데이터 건수와 최근 특보 시각을 요약한다.
- 여기서는 계산 로직을 만들지 않고 요약값만 보여준다.

### 추천 `pages/1_대피소_추천.py`

- 기존 특보 CSV 기준으로 최근 특보와 기본 재난 목록을 섞어 사용자 선택 옵션을 만든다.
- 재난 유형에 따라 전용 대피소를 먼저 찾고, 부족하면 통합 대피소를 fallback으로 붙인다.

### 실시간 `pages/2_실시간_테스트.py`

- `realtime_support.py`의 위치 입력, OSRM 경로, 지도 helper를 사용한다.
- 추천 계산 자체는 `pages/1_대피소_추천.py`의 helper를 그대로 재사용한다.

### 재난문자 안내 `pages/4_재난문자_대피_안내.py`

- `crawler_alerts_data.py`로 최근 재난문자를 읽는다.
- `지역 -> 시군구정규화` 우선 규칙으로 현재 위치와 맞는 최근 재난문자를 고른다.
- 선택된 재난문자의 `재난그룹`을 이용해 기존 추천 로직으로 대피소를 추천한다.
- 경로 계산과 지도 생성은 `realtime_support.py`를 재사용한다.

### 분석 `pages/3_Data_Analysis.py`

- 기존 특보 CSV를 다시 분석용 DataFrame으로 만들고 `재난그룹` 컬럼을 붙인다.
- 추천 페이지처럼 한 지역의 결과를 좁게 보는 대신 전체 분포와 추이를 보는 쪽에 집중한다.
