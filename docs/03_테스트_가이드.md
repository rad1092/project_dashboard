# 테스트 가이드

## 기본 실행

```powershell
.\.venv\Scripts\python -m pytest -q
```

## 왜 `PROJECT_DASHBOARD_IMPORT_ONLY=1`를 쓰는가

테스트는 페이지를 실제로 띄우는 것이 목적이 아니라, 각 파일 안 helper 함수가 기대한 값을 만드는지 확인하는 것이 목적이다.

그래서 `tests/conftest.py`는 모듈을 import할 때 `PROJECT_DASHBOARD_IMPORT_ONLY=1`를 먼저 설정한다. 이 값이 있으면 `app.py`와 `pages/*.py`는 import만 되고 `render_page()`는 자동 실행되지 않는다.

## 샘플 데이터 fixture

`sample_preprocessing_dir` fixture는 임시 폴더 안에 최소한의 CSV를 만든다.

- `preprocessing/danger_clean.csv`
- `preprocessing/final_shelter_dataset.csv`
- `preprocessing/earthquake_shelter_clean_2.csv`
- `preprocessing/tsunami_shelter_clean_2.csv`
- `preprocessing_code/data/disaster_message_realtime.csv`

이 fixture 덕분에 테스트는 실제 운영 데이터 폴더가 없어도 추천 규칙, 크롤러 로더, 실시간 경로 로직을 검증할 수 있다.

## 현재 테스트가 보는 것

- `tests/test_home_page.py`
  - 홈 메타데이터, KPI, 데이터 카탈로그가 현재 구조와 맞는지 확인한다.
- `tests/test_recommendation_page.py`
  - 데이터 경로 우선순위, 지역 추정, 재난 분류, 추천 규칙, fallback 동작을 확인한다.
- `tests/test_realtime_page.py`
  - `realtime_support.py`의 OSRM 응답 정리, fallback route, 지도 생성과 Page 2 공유 상태를 확인한다.
- `tests/test_crawler_alerts_data.py`
  - 크롤링 재난문자 CSV 로더와 재난그룹 매핑, 최근 재난문자 조회 규칙을 확인한다.
- `tests/test_message_guidance_page.py`
  - Page 4 import 구조와 크롤링 재난문자 기반 추천 연결을 확인한다.
- `tests/test_analysis_page.py`
  - 분석용 DataFrame과 KPI, 차트 빌더가 빈 데이터 없이 동작하는지 본다.
- `tests/test_static_pages.py`
  - 문서 파일 존재 여부와 페이지 파일 compile 가능 여부를 확인한다.

## 환경 이슈를 먼저 점검해야 하는 경우

테스트 코드가 아니라 로컬 파이썬 환경 문제로 실패할 때가 있다. 예를 들어 `numpy`나 `pandas` DLL 로딩 오류가 나면 테스트 로직 전에 멈춘다.

이 경우에는 코드 수정보다 먼저 아래를 확인한다.

- `.venv`가 깨지지 않았는지
- Windows 권한이나 백신이 DLL 로딩을 막지 않는지
- `uv sync` 또는 가상환경 재생성이 필요한지
