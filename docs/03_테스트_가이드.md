# 테스트 가이드

## 기본 실행

```powershell
.\.venv\Scripts\python -m pytest -q
```

## 왜 `PROJECT_DASHBOARD_IMPORT_ONLY=1`를 쓰는가

테스트 목적은 페이지를 실제로 띄우는 것이 아니라, 페이지 모듈과 helper가 import 가능한지 확인하는 데 있다.

그래서 `tests/conftest.py`는 모듈 import 전에 `PROJECT_DASHBOARD_IMPORT_ONLY=1`를 설정한다.
이 값이 있으면 `app.py`와 `pages/*.py`는 import만 되고 `render_page()`는 자동 실행되지 않는다.

## 현재 테스트 범위

- `tests/test_home_page.py`
  - HOME 메타 구성과 KPI 계산을 확인한다.
- `tests/test_realtime_page.py`
  - `pages/1_대피_안내_시뮬레이션.py`의 경로 계산과 fallback 동작을 확인한다.
- `tests/test_message_guidance_page.py`
  - `pages/2_실시간_대피_안내.py`가 최신 재난문자 기준으로 추천을 만드는지 확인한다.
- `tests/test_mock_disaster_message.py`
  - 모의 재난문자 생성기가 원본 스키마와 `page 2` 로더에 맞는 CSV를 쓰는지 확인한다.

## 모의 재난문자 생성

```powershell
.\.venv\Scripts\python preprocessing_code\mock_disaster_message.py --sido 경북 --sigungu 포항시
```

위 명령은 `preprocessing_code/data/disaster_message_realtime.csv`를 덮어써서 `pages/2_실시간_대피_안내.py`가 바로 읽을 수 있는 모의 재난문자 1건을 만든다.
- `tests/test_analysis_page.py`
  - `pages/3_데이터_분석.py`의 필터, KPI, 8개 차트 builder를 확인한다.
- `tests/test_static_pages.py`
  - 문서 존재 여부와 주요 페이지 파일 compile 가능 여부를 확인한다.

## 환경 메모

로컬 환경에서 `numpy` 또는 `pandas` DLL 로드 오류가 나면 테스트가 코드 이전에 멈출 수 있다.
이 경우 `.venv` 상태와 Windows 권한 문제를 먼저 확인한다.
