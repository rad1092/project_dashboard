# 작업 체크리스트

## 이 체크리스트를 쓰는 방법

작업을 시작하기 전, 코드 작성 중, 마무리 전에 각각 빠르게 훑어보는 용도다.
초보자라면 "기능을 바꾼다"는 생각보다 "어느 층의 약속을 건드리는가"를 먼저 확인하는 데 도움이 된다.

## 작업 전

- [ ] 이번 변경이 페이지 수정인지, 추천 규칙 수정인지, 데이터 경로 수정인지 구분했는가?
- [ ] 관련 서비스 파일과 관련 docs 를 먼저 확인했는가?
- [ ] 외부 CSV 를 직접 수정하려는 작업이 아닌가?
- [ ] 이번 변경에서 `감지 지역`, `활성 지역`, `직선 거리` 같은 핵심 용어 의미가 바뀌는가?

## 코드 작성 중

- [ ] 페이지 파일이 CSV 로딩과 추천 규칙까지 직접 떠안지 않는가?
- [ ] 유료 API 나 실시간 API 호출 코드를 현재 단계 코드에 넣지 않았는가?
- [ ] 미래 확장 코드는 실제 실행 코드와 분리되어 있는가?
- [ ] 새 함수와 새 파일에 설명 주석을 충분히 달았는가?
- [ ] 초보자가 막힐 만한 `session_state`, `copy()`, `groupby()`, fallback 분기 지점에 필요한 인라인 주석이 있는가?

## 마무리 전

- [ ] `pytest` 를 통과했는가?
- [ ] 추천 페이지와 분석 페이지가 실제로 열리는가?
- [ ] README 와 docs 설명이 현재 코드와 같은 이야기를 하는가?
- [ ] 외부 전처리 CSV 는 그대로 유지했는가?
- [ ] 현재 구현과 미래 확장을 섞어 쓰는 문구가 새로 들어가지 않았는가?
