# 내보내기 가이드

## 이 문서의 목적

현재 저장소 내용을 Word, PDF, PPT 같은 다른 형식으로 옮길 때
무엇을 어떤 순서로 설명하면 좋은지 정리한 문서다.

## 문서로 옮길 때

현재 저장소의 기준 원본은 Markdown 과 Python 주석이다.
Word, PDF, PPT 로 옮길 때는 아래 순서를 권장한다.

1. 프로젝트 목적과 제약
2. 페이지 구조
3. 추천 플로우 설명
4. 감지 지역과 활성 지역 개념
5. 데이터셋 구성
6. 분석 화면 요약
7. 미래 실시간 확장 계획

특히 발표용이든 문서용이든 아래 문장은 빠뜨리지 않는 편이 좋다.

- 현재는 과거 전처리 데이터 기반으로 동작한다.
- 현재 지역 감지는 근접 추정이며, 행정경계 기반 역지오코딩이 아니다.
- 거리 기준은 직선 거리이며 실제 경로 안내는 아직 제공하지 않는다.

## 발표 자료로 옮길 때

- `2 대피소 추천` 화면: 현재 동작 예시
- `3 작동 설명` 화면: 플로우 설명
- `4 실시간 준비` 화면: 향후 확장 계획
- `6 Data Analysis` 화면: 근거 데이터 요약

이미지 한 장보다 현재 페이지 흐름 자체를 캡처해 설명하는 편이 좋다.
특히 `2 -> 3 -> 4` 순서로 보여 주면 "현재 구현 -> 근거 설명 -> 미래 확장" 흐름이 자연스럽다.
